介绍

PHP (PHP Hypertext Preprocessor)的优势:

  • 跨平台兼容
  • 兼容几乎所有服务器
  • 可以支持大多数的数据库
  • 开源免费
  • 文件以.php结尾

PHP能做什么:

  • 生成动态网页
  • 在服务端可以对文件进行多种处理
  • 可以收集信息和数据
  • 可以收发cookies
  • 可以修改维护数据库
  • 可以加密信息

安装

由于PHP是服务器语言, 所以开发者必须要有一个服务器或者本地服务器才能运行PHP语言.

  • 安装一个web server
  • 安装PHP
  • 安装数据库 (e.g.MySQL)

PHP Variables

与别的语言不同, PHP声明variable的时候需要在之前添加符号$

<?php
$txt = "Hello world!";
$x = 5;
$y = 10.5;
?>

variable的命名有要求

  • variable name只能以字母或下划线_开头, 不能以数字开头
  • variable name可以含有数字, 字母和下划线
  • variable name是case sensitive的, 也就是说$hello与$HELLO是两个不同的variable

$txt = "linghang.education";

echo $x + $y;
?>

PHP Variables Scope

由于PHP可以在一个文档的任意位置, 所以variable的scope就变得很重要了

Global and Local Scope

一个在function外部声明的variable有Global Scope, 可以且仅可以在function外部执行调用.

<?php
$x = 5; // global scope

function myTest() {
    // using x inside this function will generate an error
    echo "<p>Variable x inside function is: $x</p>";
} 
myTest();

echo "<p>Variable x outside function is: $x</p>";
?>

在function内部声明的variable有Local Scope, 可以且仅可以在funciton内部执行

<?php
function myTest() {
    $x = 5; // local scope
    echo "<p>Variable x inside function is: $x</p>";
} 
myTest();

// using x outside the function will generate an error
echo "<p>Variable x outside function is: $x</p>";
?>

PHP The global Keyword

global是PHP中的关键字, 可以让开发者在function内部调用外部的variable

<?php
$x = 5;
$y = 10;

function myTest() {
    global $x, $y;
    $y = $x + $y;
}

myTest();
echo $y; // outputs 15
?>

PHP会把所有的global variable都储存在一个叫$GLOBALS[*index*]的Array中, 其中的index就是variable name. 所以在function内部调用这个Array的话, 不但可以取到variable的值, 还可以对其进行修改

<?php
$x = 5;
$y = 10;

function myTest() {
    $GLOBALS['y'] = $GLOBALS['x'] + $GLOBALS['y'];
} 

myTest();
echo $y; // outputs 15
?>

PHP The static Keyword

在PHP function声明的variable, 在function运行完之后就会被销毁, 但有时候开发者不希望这个variable被销毁, 那么可以用static这个关键字

<?php
function myTest() {
    static $x = 0;
    echo $x;
    $x++;
}

myTest();
myTest();
myTest();
?>

PHP echo and print Statements

在PHP中有两种输出方法, echo和print, 这两个不是函数, 而是语言结构, 用法上基本相同,
最重要的不同点是:

  • echo可以输出多个字符串, 而print只可以输出一个字符串
  • print可以表现得像一个函数, 比如: $result = print "Hello World";
  • print()返回值, 成功返回1, 否则返回0. 而echo()没有返回值
  • echo相对速率比较快, 因为没有返回值

PHP Switch Statement

switch Statement是if statement的简写形式
Switch会根据条件来选择进入哪个case, 然后运行到break后跳出statement, 如果都没有满足的条件, 则会运行default

Syntax

switch (n) {
    case label1:
        code to be executed if n=label1;
        break;
    case label2:
        code to be executed if n=label2;
        break;
    case label3:
        code to be executed if n=label3;
        break;
    ...
    default:
        code to be executed if n is different from all labels;
}

PHP for Loop

for loop的条件判定是由三部分组成的

  • 初始化: 初始化loop counter的值
  • 判定: 判定是否满足循环的条件
  • 结束句: 当运行完一次loop后对loop counter进行处理

Syntax

for (init counter; test counter; increment counter) {
    code to be executed;
}

PHP中POST, GET, REQUEST的区别

  • $_REQUEST[]具用$_POST[] $_GET[]的功能，但是$_REQUEST[]比较慢。通过POST和GET方法提交的所有数据都可以通过$_REQUEST数组获得。
  • GET是从服务器上获取数据，POST是向服务器传送数据。
  • GET是把参数数据队列加到提交表单的ACTION属性所指的URL中，值和表单内各个字段一一对应，在URL中可以看到。POST是通过HTTP POST机制，将表单内各个字段与其内容放置在HTML HEADER内一起传送到ACTION属性所指的URL地址。用户看不到这个过程。
  • 对于GET方式提交表单数据，服务器端用$_GET[‘name’]获取变量的值，对于POST方式提交表单数据，服务器端用$_POST[‘name’]获取提交的数据，当然，两者都可以通过$_REQUEST[‘name’]获得表单数据。对于REQUEST方式提交表单数据，服务器端用$_REQUEST[‘name’]获取变量的值，但这种方式很少用。
  • GET传送的数据量较小，不能大于2KB。POST传送的数据量较大，一般被默认为不受限制。但理论上，一般认为不能超过100KB。
  • GET安全性非常低，POST安全性较高。
  • GET表单值可以通过_GET获取；但通过action的url设置的参数总是获取不到的，<form method="get" action="a.asp?b=b">跟<form method="get"action="a.asp">是一样的，也就是说，在这种情况下，GET方式会忽略action页面后边带的参数列表。POST表单值可以通过_POST获取；但通过action的url参数设置的参数则可以不能通过_POST获取到。action=test.php?id=1这种就是GET方式传值，可以用$_REQUEST和$_GET接受传值，但不能用POST方式获取到值，即使表单是POST方式提交。所以，在提交表单时，如果action中同时有参数，最好只能通过POST表单方式，对于表单内数据，直接通过POST获取，对于action中参数，通过GET获取。

PHP include与require的区别

include与require都是用来加在外部文件的方法, 区别在于:

  • 如果加在失败:
    • include会产生一个warning, 但是脚本会继续执行
    • warning会爆出一个fatal error, 而脚本会停止执行
    • 可以使用require_once()和include_once()检测文件是否重复包含
  • 对于PHP的性能来说:
    • include执行文件的时候, 每次都需要进行读取和评估
    • require对于文件只处理一次
    • 所以对于需要多次执行一段代码, 使用require的效率比较高.
    • 而如果每次执行代码的文件不同, 或者需要对一个文件的代码进行叠代的循环, 就要使用include
  • 使用方式也不同:
    • 对于require来说, 一般性是放在PHP 文件的最前面, 执行的时候就会先读入require所指定的文件, 变成PHP程序的一部分
    • 对于include来说, 一般是放在流程控制的处理区段中, PHP在执行到include的时候, 才将其文件读出来, 这样可以使流程简单化.

这在遍历一个长度不确定的文档是很有用的.

<?php
$myfile = fopen("webdictionary.txt", "r") or die("Unable to open file!");
// Output one line until end-of-file
while(!feof($myfile)) {
  echo fgets($myfile) . "<br>";
}
fclose($myfile);
?>

PHP Read Single Character - fgetc()

fgetc()用来读文件的一个字节

<?php
$myfile = fopen("webdictionary.txt", "r") or die("Unable to open file!");
// Output one character until end-of-file
while(!feof($myfile)) {
  echo fgetc($myfile);
}
fclose($myfile);
?>

PHP Write to File - fwrite()

当一个文件已经被打开, 同时可写的时候, 开发者可以使用fwrite() 往文件里面写内容
fwrite()包含两个参数: 第一个是写入的文档名字, 第二个是写入的内容
开发者要注意的是, fopen()里面定义了文件打开的方式, 包括了是擦除所有内容后重写还是在原有内容的结尾继续写. 而fwrite()则只负责按照之前的打开方式写入内容

<?php
$myfile = fopen("newfile.txt", "w") or die("Unable to open file!");
$txt = "John Doe\n";
fwrite($myfile, $txt);
$txt = "Jane Doe\n";
fwrite($myfile, $txt);
fclose($myfile);
?>

PHP5 File Upload

PHP 中上传文件是很容易的, 但是开发者必须很小心用户上传的文件有可能会对数据和服务器造成攻击

Configure The “php.ini” File

首先, 开发者要在php.ini文件中允许用户上传文件.

file_uploads = On

Create The HTML Form

注意:

  • form的提交要用method="post"
  • form还可以添加一些别的属性: enctype="multipart/form-data", 这具体说明了提交form的时候用了哪一种编码形式
  • type="file"是用来上传文件的, 会在输入框边上生成一个选择文件的"Browse"按钮
  • 提交后的form信息会被存储到upload.php文件中.
<!DOCTYPE html>
<html>
<body>

<form action="upload.php" method="post" enctype="multipart/form-data">
    Select image to upload:
    <input type="file" name="fileToUpload" id="fileToUpload">
    <input type="submit" value="Upload Image" name="submit">
</form>

</body>
</html>

Create The Upload File PHP Script

$target_dir = "uploads/"声明了文件上传后存储的位置
$target_file声明了上传文件在用户本地的路径
$imageFileType存储了上传文件的文件扩展名

<?php
$target_dir = "uploads/";
$target_file = $target_dir . basename($_FILES["fileToUpload"]["name"]);
$uploadOk = 1;
$imageFileType = pathinfo($target_file,PATHINFO_EXTENSION);
// Check if image file is a actual image or fake image
if(isset($_POST["submit"])) {
    $check = getimagesize($_FILES["fileToUpload"]["tmp_name"]);
    if($check !== false) {
        echo "File is an image - " . $check["mime"] . ".";
        $uploadOk = 1;
    } else {
        echo "File is not an image.";
        $uploadOk = 0;
    }
}
?>

Check if File Already Exists

在上传文件前, 开发者可以先检查用户是否已经上传了一样的文件到服务器的upload文件夹中. 如果已经上传了, 那么显示提示信息, 同时$uploadOk设为0.

// Check if file already exists
if (file_exists($target_file)) {
    echo "Sorry, file already exists.";
    $uploadOk = 0;
}

Limit File Size and Type

同时, 开发者还可以给于上传文件大小的限制和文件的类型, 如果超出限制或者文件类型不符合, 那么显示提示信息, 同时$uploadOk设为0.

 // Check file size
if ($_FILES["fileToUpload"]["size"] > 500000) {
    echo "Sorry, your file is too large.";
    $uploadOk = 0;
}
// Allow certain file formats
if($imageFileType != "jpg" && $imageFileType != "png" && $imageFileType != "jpeg"
&& $imageFileType != "gif" ) {
    echo "Sorry, only JPG, JPEG, PNG & GIF files are allowed.";
    $uploadOk = 0;
}

PHP5 Cookies

Cookie是用来区分用户的.
每次用户访问服务器的时候, 服务器会给用户的电脑嵌入一个文件, cookie. 当同一台电脑请求访问页面的时候, 电脑也会给服务器发送一个cookie.

Create Cookies With PHP

PHP中, 开发者可以用setcookie()来生成一个cookie

setcookie(name, value, expire, path, domain, secure, httponly);

PHP Create/Retrieve a Cookie

setcookie($cookie_name, $cookie_value, time() + (86400 * 30), "/");

  • 生成了一个叫做"user"的cookie, 值是John Doe.
  • 这个Cookie会在 30天后过期.
  • "/"表示这个cookie在整个网站都可以使用

开发者可以用$_COOKIE来检索"user"的值, 还可以用isset()来检查cookie是否被设置了.
注意: setcookie()必须在放置在<html>之前, cookie的值在上传时候会被自动的加密, 同时下载的时候也会被自动的解密. 如果要防止被加密的话, 可以使用setrawcookie().

<?php
$cookie_name = "user";
$cookie_value = "John Doe";
setcookie($cookie_name, $cookie_value, time() + (86400 * 30), "/"); // 86400 = 1 day
?>
<html>
<body>

<?php
if(!isset($_COOKIE[$cookie_name])) {
    echo "Cookie named '" . $cookie_name . "' is not set!";
} else {
    echo "Cookie '" . $cookie_name . "' is set!<br>";
    echo "Value is: " . $_COOKIE[$cookie_name];
}
?>

</body>
</html>

Delete a Cookie

PHP 5 Sessions

Session是用来提供多个页面之间传输数据的方法.
与Cookie不同的是, session不会存储在用户的电脑上.
Session的数据不会丢失 直到当用户关闭浏览器

Start a PHP Session

PHP中用seesion_start()来建立一个session, 会被存储到$_SESSION中.
注意session_start()一定要在文档的最开始, 尤其在<html>之前

<?php
// Start the session
session_start();
?>
<!DOCTYPE html>
<html>
<body>

<?php
// Set session variables
$_SESSION["favcolor"] = "green";
$_SESSION["favanimal"] = "cat";
echo "Session variables are set.";
?>

</body>
</html>

Get PHP Session Variable Values

Session不会自己传递到每个页面, 需要开发者在每个页面开头检索一下是否有session, 并且恢复session.
通常, session会在用户的电脑里面安置一个user-key, 当用户打开别的页面的时候, 它会搜寻这个user-key, 如果符合了, 就允许访问这个session, 如果不符合就会开始一个新的session.

<?php
session_start();
?>
<!DOCTYPE html>
<html>
<body>

<?php
// Echo session variables that were set on previous page
echo "Favorite color is " . $_SESSION["favcolor"] . ".<br>";
echo "Favorite animal is " . $_SESSION["favanimal"] . ".";
?>

</body>
</html>

Modify a PHP Session Variable

如果开发者要修改session, 那么只需要覆盖它即可

<?php
session_start();
?>
<!DOCTYPE html>
<html>
<body>

<?php
// to change a session variable, just overwrite it

?>

</body>
</html>

Destroy a PHP Session

如果需要移除所有的global session variables或者销毁这个session, 可以用session_unset()和session_destroy()

<?php
session_start();
?>

<body>

PHP中可以用filter来validating 和 Sanitizing data.
validating data: 用来验证是否用户输入的数据符合要求
sanitizing data: 用来审核用户输入的数据是否会对服务器造成攻击, 并且删除非法的字节

filter_list()可以用来罗列PHP filter extension.

Why Use Filters?

因为网页需要接受很多用户的输入信息, 包括:

  • 用户提交的表单
  • Cookie
  • 网页服务器的数据
  • 数据库的信息

这些外部的数据很容易被黑客利用来攻击网站, 所以用PHP Filter来验证以及净化输入的数据是很重要的

PHP filter_var() Function

filter_var() 可以用来验证和净化数据.
filter_var()包含两个参数: 第一个是开发者需要检查的variable, 第二个是用来检查的形式.

?>

PHP中开发者可以用error_function来自己定一个error handler.
error_function最少需要两个参数: error level和error message, 最大可以接受五个参数: file, line-number, error context

error_function(error_level, error_message, error_file, error_line, error_context)

PHP Exception Handling

当一个特殊的error发生了, 开发者可以用Exception来改变这个脚本的流程
当Exception被触发后:

  • 当前的代码状态会被保存
  • code exception会被切换到一个预先定义的exception handler function.
  • 根据情况, hanlder会从之前保存的代码恢复数据, 终止脚本或者继续在另一个位置运行脚本.

Basic Use of Exceptions

When an exception is thrown, the code following it will not be executed, and PHP will try to find the matching catch block.
If an exception is not caught, a fatal error will be issued with an Uncaught Exception message.

<?php
//create function with an exception
function checkNum($number) {
  if($number>1) {
    throw new Exception("Value must be 1 or below");
  }
  return true;
}

//trigger exception
checkNum(2);
?>
Fatal error: Uncaught exception 'Exception'
with message 'Value must be 1 or below' in C:\webfolder\test.php:6
Stack trace: #0 C:\webfolder\test.php(12):
checkNum(28) #1 {main} thrown in C:\webfolder\test.php on line 6

Try, throw and catch

为了避免上述例子的问题, 就需要一个适合的方法来handle an exception

  • Try - A function using an exception should be in a try block. If the exception does not trigger, the code will continue as normal. However if the exception triggers, an exception is thrown
  • Throw - This is how you trigger an exception. Each throw must have at least one catch
  • Catch - A “catch” block retrieves an exception and creates an object containing the exception information
<?php
//create function with an exception
function checkNum($number) {
  if($number>1) {
    throw new Exception("Value must be 1 or below");
  }
  return true;
}

//trigger exception in a "try" block
try {
  checkNum(2);
  //If the exception is thrown, this text will not be shown
  echo 'If you see this, the number is 1 or below';
}

//catch exception
catch(Exception $e) {
  echo 'Message: ' .$e->getMessage();
}
?>

Creating a Custom Exception Class

开发者需要生成一个特殊的class中有一个可以在exception发生时被call的function,
p.s. This class must be an extension of the exception class

<?php
class customException extends Exception {
  public function errorMessage() {
    //error message
    $errorMsg = 'Error on line '.$this->getLine().' in '.$this->getFile()
    .': <b>'.$this->getMessage().'</b> is not a valid E-Mail address';
    return $errorMsg;
  }
}

$email = "someone@example...com";

try {
  //check if
  if(filter_var($email, FILTER_VALIDATE_EMAIL) === FALSE) {
    //throw exception if email is not valid
    throw new customException($email);
  }
}

catch (customException $e) {
  //display custom message
  echo $e->errorMessage();
}
?>

Multiple Exceptions

一个脚本是可以用multipie exceptions来检测不同的情况, 可以使用if...else, switch...

<?php
class customException extends Exception {
  public function errorMessage() {
    //error message
    $errorMsg = 'Error on line '.$this->getLine().' in '.$this->getFile()
    .': <b>'.$this->getMessage().'</b> is not a valid E-Mail address';
    return $errorMsg;
  }
}

$email = "someone@example.com";

try {
  //check if
  if(filter_var($email, FILTER_VALIDATE_EMAIL) === FALSE) {

  if(strpos($email, "example") !== FALSE) {
    throw new Exception("$email is an example e-mail");

catch (customException $e) {
  echo $e->errorMessage();
}

catch(Exception $e) {
  echo $e->getMessage();
}
?>

Re-throwing Exceptions

有时候, 当一个exception被抛弃的时候, 开发者希望能用不同的方法处理它.
这个脚本应该把系统的错误对用户隐藏, 虽然系统错误很重要, 但是对于用户来说, 并不需要让用户知道有错误.

<?php
class customException extends Exception {
  public function errorMessage() {
    //error message
    $errorMsg = $this->getMessage().' is not a valid E-Mail address.';
    return $errorMsg;
  }
}

$email = "someone@example.com";

try {
  try {
    //check for "example" in mail address
    if(strpos($email, "example") !== FALSE) {
      //throw exception if email is not valid
      throw new Exception($email);
    }
  }
  catch(Exception $e) {
    //re-throw exception
    throw new customException($email);
  }
}

catch (customException $e) {
  //display custom message
  echo $e->errorMessage();
}
?>

Set a Top Level Exception Handler

set_exception_handler()是用户定义的function, 用来处理所有uncaught exceptions.

<?php
function myException($exception) {
  echo "<b>Exception:</b> " . $exception->getMessage();
}

set_exception_handler('myException');

throw new Exception('Uncaught Exception occurred');
?>

Rules for exceptions

  • 代码需要包括在try{}之间, 用来帮助捕捉潜在的exceptions
  • 每次调用try{}或者throw必须要只要有一个相对应的catch{}
  • 多次catch{}可以用来捕捉不同类的exception
  • exceptions可以在catch{}中用try{}被throw或者re-thrown

A simple rule: If you throw something, you have to catch it.

  • sort()、assort()、和 ksort() 有什么分别？它们分别在什么情况下使用？
sort()
 根据阵列中元素的值，以英文字母顺序排序，索引键会由 0 到 n-1 重新编号。主要是当阵列索引键的值无关疼痒时用来把阵列排序。
 assort()
 PHP 没有 assort() 函式，所以可能是 asort() 的笔误。
 asort()
 与 sort() 一样把阵列的元素按英文字母顺序来排列，不同的是所有索引键都获得保留，特别适合替联想阵列排序。
 ksort()
 根据阵列中索引键的值， 以英文字母顺序排序，特别适合用于希望把索引键排序的联想阵列。

  • “===” 是什么？试举一个“==”是真但“===”是假的例子。
 “===”是给既可以送回布尔值“假”，也可以送回一个不是布尔值但却可以赋与“假”值的 函式，strpos() 和 strrpos() 便是其中两个例子。

if (strpos(“abc”, “a”) == true){    
    // 这部分永不会被执行，因为 “a” 的位置是 0，换算成布尔值“假”

// 这部份会被执行，因为“===”保证函式 strpos() 的送回值不会换算成布尔值.

  • include 和 include_once 有什么分别？require 又如何？
三者都是 用来在脚本中插入其他档案，视乎 url_allow_fopen 是否核准，这个档案可以从系统内部或外部取得。但他们之间也有微细的分别：
 include()： 这个函式容许你在脚本中把同一个档案插入多次，若果档案不存在，它会发出系统警告并继续执行脚本。
 include_once()：它跟 include() 的功能相似，正如它的名字所示，在脚本的执行期间，有关档案只会被插入一次。
 require()：跟 include() 差不多，它也是用来在脚本中插入其他档案，但若果档案不存在，它会发出系统警告，这个警告会引致致命错误令脚本中止执行
